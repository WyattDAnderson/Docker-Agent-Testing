# Technical Specification for Weather Dashboard Application

## Overview and User Story
The Weather Dashboard Application provides real-time weather information for any city requested by the user. By interacting with a simple web interface, users can input a city's name and retrieve current weather data such as temperature, humidity, wind speed, and a weather icon depicting the conditions.

## Technical Architecture
- **Backend**: Node.js with Express framework.
- **Frontend**: Vanilla JavaScript for dynamic content updates.

## API Endpoints
- `GET /` - Serves the main HTML page.
- `GET /api/weather?city=X` - Fetches and returns weather data for the specified city in JSON format.
- `GET /health` - Provides health status of the API server.

## External API
- The application fetches weather data from the OpenWeatherMap API. 
- Utilize environment variable `OPENWEATHER_API_KEY` for API key;
- In absence or failure, default to a set of mock data.

## Frontend Features
- A search box for city input.
- Weather card displaying:
  - Temperature in Celsius
  - Humidity percentage
  - Wind speed
  - Weather condition icon
- Dark theme
- Responsiveness for mobile and desktop views.

## Input Validation
- City name is a required field.
- Length: 1 to 100 characters.
- Sanitization to prevent XSS attacks.

## Error Handling
- **400**: Bad Request for invalid user input.
- **404**: City not found in the weather data.
- **500**: Internal Server Error for API failures.

## Testing
- Implement unit tests covering all modules.
- Aim for at least 80% code coverage.

## Dockerization
- Utilizes `node:22-alpine` as the base image.
- Implement multi-stage builds to optimize image size.
- Run application as a non-root user.
- Include a health check command to verify API server operational status.

// Handles input validation and XSS protection

function validateCity(city) {
    const result = { valid: true, error: null };
    if (typeof city !== 'string' || city.length < 1 || city.length > 100) {
        result.valid = false;
        result.error = 'City name must be a string between 1 and 100 characters';
        return result;
    }

    // Simple regex to check for script tags (rudimentary XSS protection)
    const xssPattern = /<script.*?>.*?<\/script>/gi;
    if (xssPattern.test(city)) {
        result.valid = false;
        result.error = 'City name contains invalid characters';
        return result;
    }

    // Potential for more comprehensive sanitization here
    return result;
}

// Handles fetching weather information from OpenWeatherMap API

const fetch = require('node-fetch');

const API_KEY = process.env.OPENWEATHER_API_KEY;
const BASE_URL = 'https://api.openweathermap.org/data/2.5/weather';

const mockData = {
    city: 'Mock City',
    temperature: 25,
    humidity: 50,
    windSpeed: 5,
    icon: '' // default/icon
};

async function fetchWeather(city) {
    if (!API_KEY) return Promise.resolve(mockData);

    const url = `${BASE_URL}?q=${encodeURIComponent(city)}&units=metric&appid=${API_KEY}`;

    try {
        const response = await fetch(url);
        if (!response.ok || response.status === 404) {
            throw new Error('City not found');
        }
        const data = await response.json();

        return {
            city: data.name,
            temperature: data.main.temp,
            humidity: data.main.humidity,
            windSpeed: data.wind.speed,
            icon: data.weather[0].icon
        };
    } catch (error) {
        console.error('Error fetching weather:', error);
        return mockData;
    }
}

module.exports = { fetchWeather };
