As a user, I want to input a city name and receive real-time weather information so that I can plan my day accordingly.

- **Frontend**: A single HTML page implementing vanilla JavaScript for dynamic content handling.
- **External API**: The application will use the OpenWeatherMap API to fetch weather data.
  - Use a demo API key or an environment variable `OPENWEATHER_API_KEY` for secure access.

- **GET /** → Serves the main dashboard HTML page.
- **GET /api/weather?city=<name>** → Accepts a city name as a query parameter and returns weather data in JSON format.

- **Search Box**: Users can type in a city name to retrieve weather data.
- **Weather Display**: Includes temperature (in °C), humidity, wind speed, weather icon, and a brief description of the weather condition.
- **Error Handling**: Notify users when an invalid city is entered.
- **UI/UX**: Clean, modern interface incorporating a preferred dark theme, leveraging CSS for styling.
- **Responsive Design**: Must be functional and visually appealing on both desktop and mobile devices.

## Input Validation
- **City Name**: Input is required, must be a string of 1-100 characters, and can contain alphanumeric characters with spaces.
- **Security**: Implement sanitization against XSS attacks.

- **Client Errors**: Return a 400 status code for invalid inputs.
- **Not Found**: Return a 404 status code if the city is not found.
- **Server Errors**: Return a 500 status code for API failures with graceful error messages to the frontend.

- **Coverage Goal**: Aim to achieve test coverage above 80%.
